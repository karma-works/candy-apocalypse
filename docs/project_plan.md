# Candy Apocalypse - Project Plan

**Core Development Philosophy**:

- **Verifiable Progression**: Each phase MUST lead to a working, playable project with an automatically verifiable outcome.
- **Testing**: We will implement robust Unit Tests (e.g., using Vitest) and End-to-End (E2E) testing (e.g., using Playwright) to verify rendering and game logic.
- **Asset Generation & Atmosphere**: All SVG assets, textures, and sprites will be generated by the AI assistant (potentially via sub-agents). **All visual assets MUST adhere to the styleguide in [Product Design](./product_design.md)**, specifically the "Candy Apocalypse" color theme, comic-style outlines, and joyful aesthetic. The original dark Doom atmosphere is replaced with our happy, cartoon vision.
- **Full Game Scope**: Our goal is a fully working game. We will implement or adapt from doom.ts and Freedoom: Sound, Music, Levels, and structural Assets.
